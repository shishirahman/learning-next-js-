
**Step 1: প্রজেক্ট সেটআপ (Setup)**

প্রথমে আপনার কম্পিউটারে একটা ফোল্ডার বানিয়ে টার্মিনাল ওপেন করুন এবং নিচের কমান্ডটি দিন। (Next.js এর লেটেস্ট ভার্সন ইন্সটল হবে)।

```bash
npx create-next-app@latest my-app
```

ইন্সটল করার সময় কিছু প্রশ্ন করবে, আপনি নিচের মত উত্তর দিবেন:
*   TypeScript: **Yes** (ভালো ডেভেলপার হতে হলে এটা লাগবে, ভয় পাওয়ার কিছু নাই)
*   ESLint: **Yes**
*   Tailwind CSS: **Yes**
*   `src/` directory: **Yes** (কোড গুছিয়ে রাখার জন্য ভালো)
*   App Router: **Yes** (এটাই আমরা শিখব)
*   Turbopack: **No** (এখনো বেটা স্টেজে আছে, দরকার নাই)
*   Import alias: **Yes** (ডিফল্ট যেটা আছে `@/*` সেটাই থাক)

---

**Step 2: ফোল্ডার স্ট্রাকচার বোঝা**

ইন্সটল শেষ হলে ফোল্ডারটি VS Code এ ওপেন করুন। `src/app` ফোল্ডারের দিকে তাকান। Next.js এ এই `app` ফোল্ডারটাই সব।

এখানে দুটি মেইন ফাইল দেখবেন:
1.  **page.tsx**: এটা হলো আপনার ওয়েবসাইটের হোমপেজ (`/` route)। রিয়েক্টের `App.js` এর মত ভাবতে পারেন।
2.  **layout.tsx**: এটা হলো আপনার অ্যাপের মেইন কাঠামো (Header/Footer এখানে থাকে)।

---

**Step 3: আসল খেলা - Server Component vs Client Component**

এখানেই রিয়েক্ট ডেভেলপাররা ভুল করে। খুব মনোযোগ দিয়ে পড়ুন।

**Standard React (Vite/CRA):**
আপনি যখন রিয়েক্টে কোড করেন, পুরো জাভাস্ক্রিপ্ট বান্ডিল ইউজার এর ব্রাউজারে যায়, তারপর সেখানে রেন্ডার হয়। একে বলে **Client Side Rendering (CSR)**।

**Next.js (App Router):**
Next.js এ ডিফল্টভাবে সব কম্পোনেন্ট **Server Component**। অর্থাৎ, আপনার কম্পোনেন্ট সার্ভারে (Back-end) রেন্ডার হয়ে শুধু HTML হয়ে ব্রাউজারে আসবে। কোনো জাভাস্ক্রিপ্ট লোড হবে না!

**সুবিধা:** সাইট সুপার ফাস্ট হবে।
**সমস্যা:** সার্ভার কম্পোনেন্টে আপনি `useState`, `useEffect`, বা `onClick` ইভেন্ট ব্যবহার করতে পারবেন না। কারণ এগুলো ব্রাউজারের জিনিস, সার্ভারের না।

---

**Step 4: তাহলে `useState` বা `onClick` কখন ইউজ করব?**

যখনই আপনার ইন্টারেক্টিভিটি দরকার (যেমন: বাটন ক্লিক, ফর্ম ইনপুট, স্টেট চেঞ্জ), তখন আপনাকে ওই কম্পোনেন্টটাকে **"Client Component"** বানাতে হবে।

এটা করার নিয়ম খুব সিম্পল। ফাইলের একদম উপরে একটা লাইন লিখতে হয়:
`"use client"`

---

**Step 5: কোড উদাহরণ (Code Example)**

নিচে দুটি উদাহরণ দিচ্ছি, পার্থক্যটা দেখুন।

**১. Server Component (ডিফল্ট - যখন ডাটা দেখানো বা স্ট্যাটিক কন্টেন্ট এর কাজ):**
(এখানে `console.log` দিলে সেটা ব্রাউজারের কনসোলে না এসে, আপনার VS Code এর টার্মিনালে আসবে, কারণ এটা সার্ভারে রান হচ্ছে!)

```tsx
// src/app/page.tsx
export default function HomePage() {
  console.log("আমি সার্ভারে রান হচ্ছি!"); // এটা ব্রাউজারে দেখা যাবে না

  return (
    <div className="p-10">
      <h1 className="text-3xl font-bold">হ্যালো Next.js!</h1>
      <p>এটা সার্ভার থেকে সরাসরি আসছে।</p>
    </div>
  );
}
```

**২. Client Component (যখন বাটন বা স্টেট দরকার):**
আমাদের যদি একটা কাউন্টার বানাতে হয়, তাহলে `"use client"` লাগবে।

```tsx
// src/app/counter.tsx
"use client"; // এই লাইনটাই ম্যাজিক! এটা না দিলে এরর খাবেন।

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button 
      className="bg-blue-500 text-white px-4 py-2 rounded"
      onClick={() => setCount(count + 1)}
    >
      Count is {count}
    </button>
  );
}
```

**আপনার আজকের কাজ (Homework):**

১. `create-next-app` দিয়ে প্রজেক্ট ইন্সটল করুন।
২. `src/app/page.tsx` ফাইলটি ক্লিন করে সেখানে নিজের নাম এবং একটা `<h1>` ট্যাগ দিন।
৩. `src/app` ফোল্ডারে `components` নামে একটা ফোল্ডার বানান। সেখানে `Button.tsx` নামে একটা ফাইল খুলুন।
৪. ওই `Button.tsx` এ একটা বাটন বানান যাতে ক্লিক করলে `alert("Hello World")` দেখাবে।
৫. মনে রাখবেন, যেহেতু `onClick` ইভেন্ট আছে, তাই `Button.tsx` এর উপরে `"use client"` লিখতে হবে।
৬. সবশেষে `page.tsx` এর মধ্যে ওই বাটনটাকে ইম্পোর্ট করে দেখান।
